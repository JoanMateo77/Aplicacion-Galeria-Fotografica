NOTAS IMPORTANTES
================================================================================

1. El proyecto sigue Clean Architecture con separación clara de capas:
   - Domain: Lógica de negocio independiente de Android
   - Infrastructure: Implementaciones concretas (Room, CameraX, etc.)
   - Presentation: UI con Jetpack Compose
   - Application: Configuración de Hilt

2. Se utiliza Hilt para inyección de dependencias en todas las capas.

3. Room se usa para persistencia local de fotos.

4. CameraX se integra para funcionalidad de cámara.

5. Accompanist Permissions está marcado como deprecated - considerar migración
   a APIs nativas de permisos de Compose.

6. La estructura de datos locales (Room) está organizada de forma limpia:
   - dao/ contiene PhotoDao.kt (DAO de Room)
   - entity/ contiene PhotoEntity.kt (Entidad de Room)
   - Convención de nombres en singular para mantener consistencia
